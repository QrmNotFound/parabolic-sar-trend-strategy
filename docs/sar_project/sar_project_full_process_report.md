## 1. 项目背景与重做动机

SAR，即 Parabolic Stop and Reverse，是一种趋势跟踪技术指标。它的基本思想是：在价格处于上升趋势时，SAR 位于价格下方并作为趋势跟踪止损参考；当价格跌破 SAR 时，趋势可能反转；在下降趋势中，SAR 位于价格上方，当价格突破 SAR 时，趋势可能转强。

原本科项目试图使用 SAR 指标构建股票交易策略，但原报告中的代码和结论之间存在较大断裂，尤其是样本外测试、参数优化和交易约束部分。对于课程报告而言，单纯给出一个较高收益数字并不能体现研究能力；相反，能够识别回测偏差、重建数据流程、明确交易假设、真实记录结果和承认局限，才更符合券商研究所、金融工程或量化研究支持实习对候选人的能力期待。

因此，本项目重做的核心目标不是“证明 SAR 一定有效”，而是完成一次可复现、可解释、可审计的策略研究流程。

## 2. 原 SAR 报告的主要问题

### 2.1 样本外测试没有真正运行

原代码中样本外收益率和夏普比率直接写入固定数值，而不是由样本外区间真实回测得到。这会导致报告结论缺乏证据支持。

错误示例的含义可以概括为：

```python
sample_out_return = 0.15
sample_out_sharpe = 1.2
```

这类写法只能说明作者希望报告显示某个结果，不能说明策略真的在样本外取得了对应表现。

### 2.2 参数优化使用随机结果

原报告参数优化部分使用随机生成的夏普比率，再从随机值中选择最大值。这无法证明最优参数有效，也无法证明参数稳定。

错误逻辑可以概括为：

```python
for params in parameter_grid:
    sharpe = 1.0 + random_noise
select params with max(sharpe)
```

正确做法应当是对每组参数真实运行回测，并在样本内区间选择参数，再将固定参数应用到样本外区间。

### 2.3 股票池存在生存者偏差

原项目使用 2024 年沪深300成分股名单中的前10只股票回测 2014-2024 年历史表现。这等于提前知道哪些股票在 2024 年仍然留在指数中，可能排除了历史上被调出、退市或表现较差的股票，从而高估策略表现。

本项目改用每月沪深300历史权重前50只股票，并在每个交易日只使用最近已知的历史成分数据，以降低生存者偏差。

### 2.4 胜率定义错误

原报告把“组合日收益为正的交易日比例”解释为“盈利交易次数占比”。这两个概念不同：

- 正收益日比例：组合当天净值上涨的天数占比。
- 逐笔交易胜率：每笔完整卖出交易中盈利交易占比。

本项目分别计算 `positive_day_ratio` 和 `trade_win_rate`，避免混淆。

### 2.5 缺少真实交易约束

原报告没有完整纳入 A 股交易约束，例如：

- t 日收盘信号不能在 t 日收盘价成交；
- A 股股票最小买入单位为 100 股；
- 买卖需要支付佣金；
- 卖出需要缴纳印花税；
- 成交价格可能存在滑点；
- 涨停开盘可能无法买入；
- 跌停开盘可能无法卖出；
- 停牌或缺少开盘价时不能成交。

本项目在回测引擎中显式实现上述约束。

策略使用复权后的开盘、最高、最低、收盘价格计算指标和成交价格。复权逻辑如下：

```text
adjusted_price = raw_price * adj_factor / latest_adj_factor
```

项目中生成的核心字段包括：

| 字段 | 含义 |

股票数据、指数数据和交易日历使用 `trade_date` 对齐。若某只股票在某一交易日缺少开盘价，则该日不能成交。若持仓股票当日缺少收盘价，则组合估值使用该持仓成本价作为保守替代。

## 6. 技术指标与交易信号

### 6.1 SAR 指标

SAR 指标用于判断趋势方向。项目使用复权最高价和复权最低价计算 SAR。最终入选参数为：

| 参数 | 数值 |
| --- | ---: |
| SAR acceleration | 0.003 |
| SAR maximum | 0.05 |

这组参数明显低于常见默认值 `0.02/0.20`，属于慢速 SAR。慢速 SAR 的优点是降低趋势反复切换带来的噪声交易，缺点是反应较慢，可能在趋势反转初期承受更大回撤。

### 6.2 RSI 指标

RSI 用于过滤过热状态。项目计算 14 日 RSI。最终参数中 `rsi_ceiling = 100`，意味着最终策略实际上不主动使用 RSI 上限过滤买入，而是保留 RSI 计算和接口，以便参数网格中可扩展。

这并不代表 RSI 无意义，而是样本内约束选择后，当前数据下 RSI 过滤未成为最优条件。报告需要如实说明这一点。

### 6.3 成交量比例

成交量比例定义为：

```text
volume_ratio = 当日成交量 / 过去20日平均成交量
```

最终参数为：

```text
volume_threshold = 0.5
```

即买入候选至少需要成交量不低于过去20日均量的一半。这个条件主要用于排除极端缩量或数据异常状态，而不是寻找放量突破。

### 6.4 买入信号

某股票在 t 日收盘后成为买入候选，需要满足：

```text
close_adj[t] > sar[t]
volume_ratio[t] > volume_threshold
rsi[t] < rsi_ceiling
股票属于 t 日动态候选池
当前未持有该股票
```

候选股票按照 `signal_strength` 排序，分数由成交量比例和 RSI 安全边际构成：

```text
signal_strength =
    0.5
  + min(volume_ratio, 3.0) / 3.0 * 0.3
  + max(rsi_ceiling - rsi, 0) / rsi_ceiling * 0.2
```

### 6.5 卖出信号

持仓股票在 t 日收盘后满足任一条件，则进入卖出队列：

```text
close_adj[t] < sar[t]
rsi[t] > rsi_ceiling
close_adj[t] < cost_basis * (1 - stop_loss)
close_adj[t] > cost_basis * (1 + take_profit)
股票离开动态候选池
```

最终参数中：

| 参数 | 数值 | 含义 |
| --- | ---: | --- |
| stop_loss | 30% | 价格相对成本下跌 30% 触发止损 |
| take_profit | 10.0 | 极宽止盈阈值，实际等于不主动止盈 |

不主动止盈的含义是：策略更依赖 SAR 趋势反转、离开股票池和止损退出，而不是在上涨后过早止盈。

## 7. 回测系统设计

### 7.1 回测原则

回测系统遵循以下原则：

1. 不使用未来数据；
2. t 日收盘后生成信号；
3. t+1 日开盘执行交易；
4. 买入按 100 股整数倍下单；
5. 买入考虑佣金和滑点；
6. 卖出考虑佣金、印花税和滑点；
7. 开盘涨停不可买入；
8. 开盘跌停不可卖出；
9. 缺少开盘价则不成交；
10. 每日记录组合净值、现金、持仓数量和换手率；
11. 每笔成交记录信号日期、成交日期、股票、方向、股数、价格、费用、收益和持仓天数。

### 7.2 交易成本参数

| 成本项目 | 参数 |
| --- | ---: |
| 买卖佣金 | 成交金额的 0.03% |
| 卖出印花税 | 成交金额的 0.10% |
| 单边滑点 | 成交价格的 0.05% |
| 最小交易单位 | 100 股 |

买入成交价：

```text
buy_price = next_open_adj * (1 + slippage_rate)
```

卖出成交价：

```text
sell_price = next_open_adj * (1 - slippage_rate)
```

### 7.3 组合构建规则

| 规则 | 参数 |
| --- | ---: |
| 初始资金 | 1,000,000 元 |
| 最大持仓数 | 30 只 |
| 调仓间隔 | 20 个交易日 |
| 单只买入资金 | 当前可用现金 / 待买入股票数 |
| 买入排序 | `signal_strength` 从高到低 |

### 7.4 为什么不能使用同日收盘成交

如果用 t 日收盘价计算信号，并假设在同一个 t 日收盘价成交，就会使用实际交易时不可得的信息。真实交易中，只有收盘后才能确认收盘价和指标状态，因此最早应在下一交易日开盘或下一可成交时点执行。本项目采用 t+1 开盘成交以降低回测偏差。

## 8. 策略伪代码

### 8.1 数据下载与缓存

```text
输入：start_date, end_date, index_code = 000300.SH

读取交易日历 trade_cal
读取每年 index_weight
对每个成分日期：
    按 weight 从高到低排序
    取前50只股票
生成历史联合股票列表

读取沪深300 index_daily

for symbol in 历史联合股票列表:
    读取 daily
    读取 adj_factor
    读取 stk_limit
    合并行情、复权因子、涨跌停价格
    生成 open_adj, high_adj, low_adj, close_adj, up_limit_adj, down_limit_adj
    缓存到 data/interim/sar_project/prices
```

### 8.2 动态股票池

```text
输入：历史成分权重表 weights, 交易日列表 trading_dates

for each 成分日期:
    universe_at_date = weight 排名前50股票

current_universe = []
for date in trading_dates:
    while 存在已公布且不晚于 date 的新成分日期:
        current_universe = 该成分日期前50股票
    universe_by_date[date] = current_universe
```

### 8.3 指标计算

```text
for symbol in 股票池:
    使用 high_adj, low_adj 计算 SAR
    使用 close_adj 计算 RSI
    使用 vol / 20日均量 计算 volume_ratio
    根据 volume_ratio 和 RSI 计算 signal_strength
```

### 8.4 每日回测主循环

```text
初始化 cash = 1,000,000
初始化 positions = {}
初始化 pending_orders = None

for index, trade_date in trading_dates:

    if pending_orders 不为空:
        在 trade_date 开盘执行上一交易日生成的订单
        卖出：
            若无开盘价：阻断
            若跌停开盘：阻断
            否则按 open_adj * (1 - slippage) 卖出
            扣佣金和印花税
        买入：
            若无开盘价：阻断
            若涨停开盘：阻断
            否则按 open_adj * (1 + slippage) 买入
            按100股向下取整
            扣佣金

    使用收盘价计算组合总市值
    记录组合净值、现金、持仓数、换手率

    使用 trade_date 收盘数据生成下一日 pending_orders：
        对已有持仓检查卖出条件
        若 index % rebalance_interval == 0:
            对候选池股票检查买入条件
            按 signal_strength 排序
```

### 8.5 参数优化

```text
参数网格：
    acceleration in [0.003, 0.005, 0.01]
    maximum in [0.05, 0.10]
    volume_threshold in [0.0, 0.5, 1.0]
    rsi_ceiling in [100]
    max_positions in [30, 50]
    rebalance_interval in [20, 60]
    stop_loss in [0.30, 1.0]
    take_profit in [10.0]

for params in 参数网格:
    在 2016-2020 样本内运行真实回测
    计算收益、超额收益、夏普、回撤、仓位、持仓数、换手率
    检查选择约束：
        average_exposure >= 45%
        average_positions >= 5
        low_exposure_day_ratio <= 35%

排序逻辑：
    先选择满足仓位/持仓约束的组合
    再检查样本内收益、超额收益和最大回撤训练底线
    对样本内累计超额收益设置打分上限
    过线后优先选择更分散的组合
    再比较样本内夏普、最大回撤和换手率

固定最优参数，在 2021-2025 样本外运行一次回测
```

## 9. 参数优化结果

样本内最终选择参数如下：

| 参数 | 数值 |
| --- | ---: |
| SAR acceleration | 0.003 |
| SAR maximum | 0.05 |
| Volume threshold | 0.5 |
| RSI ceiling | 100 |
| Max positions | 50 |
| Rebalance interval | 20 |
| Stop loss | 30% |
| Take profit | 不主动止盈 |

这组参数体现了几个重要调整：

1. 使用慢速 SAR，降低短期噪声带来的频繁交易；
2. 使用 50 只最大持仓，并在实际交易中保持约 16-18 只平均持仓，减少少数股票对组合结果的影响；
3. 使用 20 日调仓间隔，降低换手率；
4. 使用较宽止损，不因普通波动过早出场；
5. 不主动止盈，允许趋势持仓延续；
6. 不通过长期空仓制造超额收益。

### 11.2 样本外结果解释

样本外 2021-2025 年，策略累计收益率为 11.81%，同期沪深300为 -12.11%，动态前50等权组合为 -8.19%。策略相对沪深300取得正累计超额收益 23.91%，同时也高于动态前50等权组合。

策略样本外最大回撤为 -45.14%，略小于沪深300的 -45.60%。这说明策略没有明显降低深度回撤，风险控制效果有限，不能被描述为“低风险策略”。

样本外夏普比率约为 -0.04，主要原因是项目使用 3% 无风险利率，而策略年化收益率约为 2.35%。这意味着策略虽然跑赢了处于下跌期的沪深300，但绝对风险调整收益并不突出。

### 11.3 交易行为解释

样本外逐笔交易胜率为 37.74%，低于 50%，但平均盈利交易收益为 12.65%，平均亏损交易收益为 -6.18%，盈亏比约为 2.05。因此策略表现并不是依靠高胜率，而是依靠“亏损较小、盈利交易较大”的趋势跟踪特征。

样本外交易股票数为 98，单一股票成交额占比上限为 3.25%，说明策略交易分布较分散，不主要依赖单一股票成交。

### 11.4 仓位行为解释

样本外平均股票仓位为 75.97%，中位股票仓位为 83.40%，低仓位天数比例为 1.65%，零持仓日比例为 0.33%。这说明策略并非长期空仓策略。它在大多数交易日都有较高股票暴露，样本外超额收益不是简单通过规避市场风险获得。

## 15. 附录：关键代码说明

### 15.1 只读数据客户端

项目通过接口白名单限制数据客户端行为：

```python
ALLOWED_ENDPOINTS = {
    "daily",
    "adj_factor",
    "index_weight",
    "index_daily",
    "trade_cal",
    "stk_limit",
}
```

这可以避免代码在无意中调用未审计接口。token 只从环境变量或 `.env` 读取，并且快照参数会过滤 token 字段。

### 15.2 交易成本计算

```python
commission = trade_value * commission_rate
stamp_tax = trade_value * stamp_tax_rate if side == "sell" else 0.0
total_cost = commission + stamp_tax
```

该逻辑区分买入和卖出，卖出额外收取印花税。

### 15.3 100 股整数约束

```python
execution_price = raw_open_price * (1 + slippage_rate)
gross_lot_cost = execution_price * 100 * (1 + commission_rate)
lots = int(cash // gross_lot_cost)
shares = lots * 100
```

买入数量向下取整到 100 股整数倍。

### 15.4 信号和成交错位

回测主循环中，先执行上一交易日生成的 pending orders，再使用当天收盘数据生成下一批 pending orders。这保证了 t 日收盘信号不会在 t 日同价成交。

```text
for trade_date in trading_dates:
    execute(pending_orders, trade_date)
    record_portfolio_value(trade_date)
    pending_orders = make_signal_after_close(trade_date)
```

### 15.5 优化器选择约束

优化器不是简单选择收益最高参数，而是先检查策略是否有足够股票暴露：

```python
passes_constraints = (
    average_exposure >= 0.45
    and average_positions >= 5
    and low_exposure_day_ratio <= 0.35
)
```

在此基础上，优化器还要求样本内总收益为正、累计超额收益达到 10%、最大回撤不深于 -30%，并对样本内累计超额收益设置 12% 的打分上限。过线后，优先选择平均持仓数量更高、分散度更好的组合，再比较夏普、回撤和换手率。这样可以避免选择长期空仓、过度集中或训练集超额特别高但研究意义较弱的参数组合。
